Standard Chatbots Copilot / ChatGPT / Claude Web
These exercises are adapted for environments where the AI cannot execute code. You are the execution loop.
Watch for the orange callouts highlighting how to automate these flows in Agentic Tools (like Claude Code).
02Spec-Driven Building & Human-in-the-Loop TDD
Concept: Never assume generated code works. Ask the AI to write failing tests first (Red), run them yourself, and provide the error output to build the implementation (Green).
"Help me plan the development of a [Specific Feature]. First, write a brief Product Requirements Document (PRD) outlining the edge cases. Then, write the unit tests for this feature. Do not write the implementation yet. I will run these unit tests locally to ensure they fail (Red), and I will paste the error output back to you so we can write the implementation to make them pass (Green)."
⚡ The Agentic Way (Claude Code)
In a true agentic workflow, you wouldn't copy the errors back. You would type:
claude-code "Write unit tests for [Feature], execute them to ensure they fail, and then implement the feature until they pass."
The agent manages the Red/Green loop autonomously.
03Generating Manual Testing Plans
Concept: Automated tests are no replacement for manual verification. Have the AI generate scripts and curl commands for you to run and manually confirm functionality.

For APIs:

"I need to manually test this new JSON API. Generate a list of curl commands that I can copy and paste into my terminal to explore the API and verify edge cases."

For Web UIs:

"Write a standalone Python Playwright script that I can run locally to navigate to my new homepage and take screenshots, so I can manually confirm the menu is in the right place."
⚡ The Agentic Way (Claude Code)
Agentic tools have terminal access. You could simply ask Claude Code to execute the curl commands directly and summarize the JSON payloads it receives, verifying the API without leaving your editor.
04Paying Down Debt with Explanatory Artifacts
Concept: When core code becomes a "black box", ask chatbots with UI rendering (Claude Artifacts, OpenAI Canvas) to build interactive, animated visualizers of the algorithm to regain your mental model.
"Never use React in artifacts - always plain HTML and vanilla JavaScript and CSS with minimal dependencies. Build an animated explanation for how the following algorithm works: [Paste Algorithm/Code]. Include a slider for the animation which can be paused and the speed adjusted or even stepped through frame by frame while paused."
05Hoarding and Recombining Specs
Concept: The best inputs are proven examples. Combine functional snippets you already understand to build complex new systems reliably.
"This code shows how to [A]: [Paste Code A]. This code shows how to [B]: [Paste Code B]. Use these examples to put together a single HTML page with embedded HTML and CSS and JavaScript that combines these to [describe new goal]."
06Code Review & Pull Request Anti-Patterns
Concept: Never inflict unreviewed AI code on your collaborators. Use the chatbot to help document your efforts and prove you've manually verified the functionality.
"Here is the code we just finished implementing. Help me draft a comprehensive Pull Request description for this change. It should explain the high-level goal, link to our initial specifications, and include a placeholder section where I will insert my implementation notes, manual testing logs, and screenshots to prove that the code works."
